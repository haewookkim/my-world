## 개요

사용자가 학년(중1~고3)과 난이도(상·중·하)를 선택하면 시스템이 국어·수학·영어 문제를 각 1개씩 출제하고, 사용자가 답안을 입력해 제출하면 정답·해설·과목별 정오·총점을 본다. 영속성 없는 즉석 자가 점검 도구.

## 범위

### 포함

- 학년·난이도 선택 화면
- 국·수·영 1문제씩 동적 출제 (LLM 생성)
- 과목별 답안 입력 (객관식 선택지 클릭 또는 서술/단답 텍스트)
- 제출 시 사용자 답안·정답·정오·해설·과목별 획득 배점·총점 표시
- 결과 화면에서 "새 문제로 다시 풀기"(같은 학년·난이도 재출제)와 "처음으로"(선택 화면 복귀)
- 출제 실패 시 에러 메시지 + "다시 시도" (학년·난이도 입력값 유지)

### 제외

- 사용자 계정·로그인 — 싱글유저 즉석 사용 범위로 한정
- 학습 이력·결과 저장·통계 — 영속성 없음, 새로고침 시 초기화
- 배치 출제·시험지 모드 — 1조합당 3문제 1세트로 한정
- 문제 난수화 옵션, 자동 채점 고도화, 오답 노트 — `examples/requirements.md`의 확장(선택) 항목, 후속 작업
- 학년 적합성·정답 정확성에 대한 품질 보증 — LLM 출력 검증은 plan/구현 영역

## 시나리오

### 0. API 키 입력 (최초 1회)

- **Given** — 사용자가 처음 접속한 상태 (브라우저에 저장된 API 키 없음)
- **When** — 학년·난이도 선택 화면에서 "API 키" 입력란에 Gemini API 키를 입력
- **Then** — 키가 브라우저 localStorage에 저장되고, 이후 새로고침해도 다시 입력하지 않아도 된다

성공 기준:
- [ ] 키 미입력 상태에서는 "출제" 버튼이 비활성화된다
- [ ] 키 입력란에 값이 있으면 "출제" 버튼이 활성화된다
- [ ] 키 입력 후 새로고침해도 입력란에 키가 남아 있다
- [ ] 키 입력란 옆에 "도움말" 버튼이 보인다
- [ ] "도움말" 클릭 시 Gemini API 키 발급 방법(Google AI Studio 접속 → 키 생성 → 복사)이 단계별로 표시된다
- [ ] "키 삭제" 동작이 가능하다 (입력란을 비우면 localStorage에서도 제거됨)

### 1. 학년·난이도 선택 후 출제

- **Given** — 학년 선택지(중1~고3)와 난이도 선택지(상·중·하)가 보이는 시작 화면
- **When** — 학년 "중2", 난이도 "중"을 선택하고 "출제" 버튼 클릭
- **Then** — 국어·수학·영어 라벨이 붙은 카드 3개가 표시되고, 각 카드에는 문항·배점·답안 입력 영역이 포함된다

성공 기준:
- [ ] "중2" + "중" 선택 후 "출제" → "국어"·"수학"·"영어" 라벨이 붙은 카드가 정확히 3개 렌더된다
- [ ] 각 카드에 배점 표시(예: "10점")가 보인다
- [ ] 각 카드에 답안 입력 영역(객관식 선택지 또는 텍스트 입력)이 보인다
- [ ] 어떤 카드에도 정답·해설 텍스트가 보이지 않는다
- [ ] 화면 어딘가에 현재 학년·난이도("중2 / 중")가 표시된다

### 2. 모든 답안을 정답으로 제출

- **Given** — 출제된 3과목 카드가 표시된 상태
- **When** — 세 과목 답안을 모두 정답으로 입력 후 "제출" 클릭
- **Then** — 각 카드에 사용자 답안·정답·"정답" 표시·해설·획득 배점이 나타나고, 총점이 만점으로 표시된다

성공 기준:
- [ ] 세 과목 모두 정답으로 입력 후 제출 → 각 카드에 "정답" 표시가 보인다
- [ ] 각 카드에 해설 텍스트가 보인다
- [ ] 각 카드에 획득 배점이 만점과 동일하게 표시된다 (예: "10/10")
- [ ] 총점이 세 과목 만점의 합으로 표시된다 (예: "30/30")

### 3. 일부 오답으로 제출

- **Given** — 출제된 3과목 카드, 입력된 답안 중 하나 이상이 오답
- **When** — "제출" 클릭
- **Then** — 정답 과목 카드는 "정답"·획득 배점, 오답 과목 카드는 "오답"·0점·시스템 정답·해설을 표시하고, 총점은 정답 과목 배점의 합이다

성공 기준:
- [ ] 정답 과목 카드에 "정답" 표시와 만점이 보인다
- [ ] 오답 과목 카드에 "오답" 표시와 사용자가 입력한 답안, 시스템 정답이 모두 보인다
- [ ] 오답 과목 카드의 획득 배점이 0으로 표시된다
- [ ] 총점이 정답 과목들의 배점 합으로 표시된다

### 4. 빈 답안 포함 제출

- **Given** — 출제된 3과목 카드, 답안 중 하나 이상이 비어 있음
- **When** — "제출" 클릭
- **Then** — 빈 답안은 자동으로 오답(0점)으로 채점되고 정답·해설이 표시된다

성공 기준:
- [ ] 모든 답안이 비어 있어도 "제출" 버튼이 클릭 가능하다
- [ ] 비어 있던 과목 카드에 "오답" 표시와 시스템 정답·해설이 보인다
- [ ] 빈 답안 과목의 획득 배점이 0으로 표시된다
- [ ] 총점에 빈 답안 과목의 배점이 포함되지 않는다

### 5. 출제 실패

- **Given** — 학년·난이도가 선택된 시작 화면
- **When** — "출제" 클릭 후 시스템이 문제 생성에 실패함 (네트워크 오류, 잘못된 API 키, LLM 응답 오류 등)
- **Then** — 에러 메시지와 "다시 시도" 버튼이 표시되고, 학년·난이도·API 키 입력값은 유지된다

성공 기준:
- [ ] 출제 호출이 실패하면 에러 메시지(예: "출제에 실패했습니다. 다시 시도해 주세요.")가 보인다
- [ ] "다시 시도" 버튼이 보인다
- [ ] "다시 시도" 클릭 시 동일한 학년·난이도·키로 출제가 재요청된다
- [ ] 학년·난이도 선택값과 API 키가 화면에 그대로 표시되어 있다

### 6. 새 문제로 다시 풀기

- **Given** — 채점 결과가 표시된 화면
- **When** — "새 문제로 다시 풀기" 버튼 클릭
- **Then** — 같은 학년·난이도로 새로 출제된 3과목 카드가 표시되고, 이전 답안·결과·해설은 보이지 않는다

성공 기준:
- [ ] 클릭 후 새 3과목 카드가 렌더된다
- [ ] 화면의 학년·난이도 표시는 이전과 동일하다
- [ ] 이전 답안 입력값·채점 결과·해설은 더 이상 보이지 않는다
- [ ] 새 출제 호출이 실패하면 시나리오 5와 같은 에러 메시지·"다시 시도" 버튼이 표시된다

### 7. 처음으로

- **Given** — 채점 결과가 표시된 화면
- **When** — "처음으로" 버튼 클릭
- **Then** — 학년·난이도 선택 화면으로 복귀하고 이전 출제·답안·결과는 사라진다

성공 기준:
- [ ] 클릭 후 학년 선택지(중1~고3)와 난이도 선택지(상·중·하)가 다시 보인다
- [ ] 이전 출제 카드·답안·채점 결과·해설이 보이지 않는다
- [ ] 학년·난이도 선택값이 초기화된 상태이다 (선택 없음 또는 기본값)

## 불변 규칙

- **정답 보호** — 정답과 해설은 사용자가 "제출"을 트리거하기 전까지 어떤 화면 요소·DOM·네트워크 응답에도 노출되지 않는다
- **중복 호출 방지** — 출제·재출제·제출 호출이 진행 중인 동안 같은 액션을 트리거하는 버튼은 비활성화된다
- **세션 격리** — 페이지를 새로고침하면 출제·답안·결과가 모두 초기화된다 (영속 저장 없음). 단 API 키만 localStorage에 영속.
- **API 키 격리** — 사용자 API 키는 브라우저 localStorage에만 저장되고 서버에는 영구 저장되지 않는다. 매 출제 요청마다 사용자→서버→LLM 경로로 일회성 전달된다.
- **언어** — 모든 문항·해설·UI 텍스트는 한국어 (영어 과목 문항 자체는 영어 지문 포함 가능)
